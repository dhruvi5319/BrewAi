   - SearchInput: check the `<input>` wrapper has `min-h-[44px]`
   - CategoryFilter pills: already have `min-h-[44px]` — verify

5. **Remove any horizontal padding > 16px on mobile** that would push content off-screen at 390px.

Update `src/index.css`:

Add after the existing Tailwind directives and base styles:
```css
/* ─── Overflow guard — prevents horizontal scroll at any viewport ─── */
html, body {
  overflow-x: hidden;
}

/* ─── Touch target minimum — applied globally for audit compliance ─── */
/* Individual components enforce min-h-[44px] via Tailwind; this is a safety net */
@media (pointer: coarse) {
  button, a, [role="button"] {
    min-height: 44px;
    min-width: 44px;
  }
}
```

Do NOT add this inside `@layer` directives — add it after the `@tailwind utilities` line so it is not purged by Tailwind's JIT.

Also check if `prefers-reduced-motion` guard already exists in index.css (it was added in Phase 1). If the existing guard looks like:
```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after { transition: none !important; animation: none !important; }
}
```
Leave it as-is (already covers CSS transitions). If it's missing or malformed, add it.
