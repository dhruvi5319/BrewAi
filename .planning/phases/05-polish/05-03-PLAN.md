### ProductCard

Wrap the existing ProductCard's outer `<div>` in a `motion.div` using `cardVariants`. ProductCard receives its variant from the parent staggerContainer, so it should NOT manage `initial`/`animate` itself — those propagate from the parent.

From `src/lib/motion.ts`, `cardVariants` uses keys `hidden/visible/exit`. The staggerContainer also uses `hidden/visible`.

```tsx
import { motion } from 'framer-motion';
import { cardVariants, useReducedMotion } from '../../lib/motion';

export function ProductCard({ item, onCustomize, onAddToCart }: ProductCardProps) {
  const shouldReduceMotion = useReducedMotion();

  return (
    <motion.div
      variants={shouldReduceMotion ? {} : cardVariants}
      // No initial/animate here — inherited from parent staggerContainer
      layout
    >
      {/* PRESERVE existing inner div and all card content exactly as-is */}
      <div className="flex flex-col justify-between bg-surface border border-border rounded-card p-4 min-h-[160px] hover:bg-surface-raised hover:border-border-hover transition-colors duration-150 ease-in-out">
        ...
      </div>
    </motion.div>
  );
}
```

CRITICAL: Preserve ALL existing ProductCard inner content (Badge, h3 name, description, price, Button CTA). Only ADD the outer motion.div wrapper.

### MenuPage — stagger container + AnimatePresence

In `src/pages/MenuPage.tsx`, update the card grid section:

1. **Import** `staggerContainer, cardVariants, useReducedMotion` from `../../lib/motion` (or `../lib/motion` from pages dir) and `AnimatePresence, motion` from `framer-motion`.

2. **Stagger container wrapper:** Replace the grid `<div>` with a `motion.div`:
```tsx
const shouldReduceMotion = useReducedMotion();

// In the JSX where filteredItems are rendered:
<AnimatePresence mode="popLayout">
  <motion.div
    key={activeCategory + searchQuery}  // key change triggers re-animation on filter change
    variants={shouldReduceMotion ? {} : staggerContainer}
    initial={shouldReduceMotion ? false : "hidden"}
    animate="visible"
    className="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 2xl:grid-cols-4 gap-4"
  >
    {filteredItems.map(item => (
      <ProductCard
        key={item.id}
        item={item}
        onCustomize={handleCustomize}
        onAddToCart={handleAddToCart}
      />
    ))}
  </motion.div>
</AnimatePresence>
```

**Note on key strategy:** Using `key={activeCategory + searchQuery}` on the stagger container causes the entire grid to re-mount and stagger in whenever the filter changes, which satisfies F6-01 (cards animate in on category change). This is the simplest approach. The `AnimatePresence mode="popLayout"` handles smooth reflow during transitions.

3. **Verify CategoryFilter transition:** CategoryFilter already has `transition-colors duration-150` on pills. Verify the class includes `transition-colors duration-150 ease-in-out` — this satisfies F6-02 (CSS-only, no Framer Motion needed).

4. **Verify App.tsx page transitions:** Check that `App.tsx` has `AnimatePresence + pageVariants` (implemented in Phase 1, plan 01-03). It should already be there. If `pageVariants` is missing or broken, repair it to:
```tsx

    exit={shouldReduceMotion ? undefined : "exit"}

  </motion.div>
</AnimatePresence>
```

5. **Preserve all other MenuPage behavior:** loading skeleton, error state, empty state, handleCustomize, handleAddToCart. Only change the grid wrapper.
