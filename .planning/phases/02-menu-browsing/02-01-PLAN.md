<task type="auto">
  <name>Task 1: Zustand menuStore</name>
  <files>src/stores/menuStore.ts</files>
  <action>
Create src/stores/menuStore.ts implementing the MenuStore interface from TechArch § 4.2.

The store MUST implement this exact interface (copy from src/types/index.ts which Phase 1 created):

```typescript
interface MenuStore {
  items: MenuItem[];           // All items from API
  categories: string[];        // Unique categories derived from items; "All" added by component, not here
  loading: boolean;
  error: string | null;
  activeCategory: string;      // 'All' or a category name — default 'All'
  searchQuery: string;         // default ''
  filteredItems: MenuItem[];   // Derived: items filtered by activeCategory + searchQuery
  fetchMenu: () => Promise<void>;
  setCategory: (category: string) => void;
  setSearch: (query: string) => void;
  clearFilters: () => void;
}
```

Implementation rules:
1. Use `create<MenuStore>()` from zustand.
2. `fetchMenu()` calls `api.getMenu()`. Sets `loading: true` before call. On success: stores items, derives categories (unique `item.category` values from items where `item.available === true`), sets `loading: false`. On error (any throw or `response.error !== null`): sets `error: 'Could not load the menu. Please try again.'`, sets `loading: false`.
3. `categories` is derived synchronously from `items` in the `fetchMenu` success branch. Categories are unique strings sorted alphabetically. Do NOT add "All" here — the CategoryFilter component prepends it.
4. `filteredItems` is a derived value recomputed in `setCategory`, `setSearch`, `clearFilters`, and after `fetchMenu` success. Filter logic:
   - `(activeCategory === 'All' || item.category === activeCategory)`
   - AND `(item.name.toLowerCase().includes(searchQuery.toLowerCase()) || item.description.toLowerCase().includes(searchQuery.toLowerCase()))`
   - Only items where `item.available === true` are ever in `items` (API returns only available items, but double-check here).
5. `setCategory(category)`: if `category === activeCategory`, reset to `'All'` (re-click deselects). Otherwise set `activeCategory = category`. Recompute `filteredItems`.
6. `setSearch(query)`: set `searchQuery = query`. Recompute `filteredItems`.
7. `clearFilters()`: set `activeCategory = 'All'`, `searchQuery = ''`. Recompute `filteredItems` (which will be all items).
8. Import `MenuItem` from `../types/index` and `api` from `../lib/api`.
9. No raw hex colors. No hardcoded category strings.
10. Export the store as `export const useMenuStore = create<MenuStore>()(...)`.

Do NOT use `immer` middleware — plain Zustand `set()` is sufficient.

Create both filter UI components. These are pure/controlled components — they receive props and call callbacks; they do NOT call the store themselves.

**CategoryFilter.tsx:**

Props interface:
```typescript
interface CategoryFilterProps {
  categories: string[];        // e.g. ['Espresso', 'Cold Brew', 'Pour-Over', 'Tea', 'Seasonal']
  activeCategory: string;      // 'All' or a category name
  onSelect: (category: string) => void;
}
```

Render logic:
- Render an outer `<div>` with `className="flex gap-2 overflow-x-auto pb-2 scrollbar-hide"` (horizontal scroll on mobile without showing scrollbar).
- Prepend an "All" pill before the categories array. So pills are: ["All", ...categories].
- Each pill is a `<button>` with:
  - When `category === activeCategory`: `className="flex-shrink-0 px-4 py-2 rounded-pill text-sm font-semibold bg-accent text-canvas border border-accent transition-colors duration-150 ease-in-out focus-visible:ring-2 focus-visible:ring-accent focus-visible:ring-offset-2 focus-visible:ring-offset-canvas"`
  - When not active: `className="flex-shrink-0 px-4 py-2 rounded-pill text-sm font-semibold bg-surface text-secondary border border-border hover:border-border-hover hover:text-primary transition-colors duration-150 ease-in-out focus-visible:ring-2 focus-visible:ring-accent focus-visible:ring-offset-2 focus-visible:ring-offset-canvas"`
  - `onClick={() => onSelect(category)}`
  - `aria-pressed={category === activeCategory}`
  - `key={category}`
  - Min height `44px` (use `min-h-[44px]` or `py-3`): ensure touch target compliance.
- Export: `export function CategoryFilter({ categories, activeCategory, onSelect }: CategoryFilterProps)`

**SearchInput.tsx:**

Props interface:
```typescript
interface SearchInputProps {
  value: string;
  onChange: (value: string) => void;
  placeholder?: string;
}
```

Implementation:
- Use `useRef<ReturnType<typeof setTimeout> | null>(null)` for the debounce timer.
- Internal state: `const [localValue, setLocalValue] = useState(value)` — the input is controlled locally, debounce fires `onChange` after 200ms.
- On input change: update `localValue` immediately (instant visual feedback), clear previous timer, set new 200ms timer that calls `onChange(newValue)`.
- Use `useEffect` to sync external `value` prop → `localValue` when `value` changes from outside (e.g., clearFilters resets to '').
- Cleanup: clear timer in useEffect cleanup to prevent memory leaks.
- Render: a `<div className="relative">` containing:
  - A search icon (Lucide `Search` icon, 16px, positioned absolutely left-3 center-y, `text-secondary`)
  - `<input type="text" ... className="w-full pl-10 pr-4 py-3 bg-surface border border-border rounded-input text-primary placeholder:text-secondary focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-accent focus-visible:ring-offset-2 focus-visible:ring-offset-canvas transition-colors duration-150 ease-in-out hover:border-border-hover" />`
  - `placeholder={placeholder ?? 'Search drinks…'}`
  - `value={localValue}`
  - `onChange={e => { ... }}`
  - `aria-label="Search drinks"`
- Export: `export function SearchInput({ value, onChange, placeholder }: SearchInputProps)`

NO raw hex values in either file. Use only Tailwind classes mapped to design tokens.
